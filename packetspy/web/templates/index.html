Discovery Tab Guide

Overview

The Discovery tab helps you reverse-engineer unknown network protocols by grouping packets by their 2-byte opcode, analyzing payload structure, and detecting field types automatically.

Quick Start

  1. Choose an addon (or "No Addon" for raw payloads) from the first dropdown.
  2. Optionally select an Opcode File from the second dropdown to pre-label known opcodes.
  3. Click Start Session to begin collecting packets.
  4. Start a capture (or load a PCAP) from the header controls — Discovery runs alongside normal capture.
  5. The opcode table fills in as packets arrive. Each row shows an opcode, how many times it appeared, payload size, rate, and any auto-detected tags.

Addon vs Raw Mode

Addon mode (e.g. "EQ Session Protocol") decodes packets through the addon's pipeline first (CRC strip, XOR decode, decompress, defragment), then groups by the application-layer opcode. Use this when the transport layer is already handled by an addon.

Raw mode ("No Addon") groups packets by the first 2 bytes of the raw TCP/UDP payload. Use this for unknown protocols with no addon.

Opcode Table

  • Label — Click the label field on any row to name an opcode (e.g. "PlayerPosition"). Labels are saved automatically.
  • Size — Shows the payload size if fixed, or "var" if variable-length.
  • Auto-Tags — Heuristic badges like xyz-floats, string, counter, or len-prefix hint at what's inside.
  • Click a row to open Field Analysis — a byte-level structural breakdown of the payload with type guesses (float, uint16, string, etc.).

Baseline Diffing

Baseline diffing helps you figure out which opcodes correspond to a specific action:

  1. With a session running, click Start Baseline to snapshot current opcode counts.
  2. Perform an in-game action (e.g. open inventory, cast a spell, move your character).
  3. Click Show Diff to see which opcodes spiked or appeared since the baseline.
  4. Use Clear Baseline to reset and try another action.

Opcode Files

Opcode files are JSON mappings of opcode values to human-readable names. They feed both Discovery and the packet inspector across all tabs.

  • Select an opcode file from the dropdown to pre-label known opcodes. Opcodes with a known name show as placeholder text in the Label column.
  • Save your work — after labeling opcodes, click Save Opcodes to create a new opcode file you can reuse in future sessions.
  • Switching files updates opcode labels everywhere — packet detail views, Discovery, and stream follow views.
  • Opcode files live in opcode_files/<addon_id>/ as JSON. You can also edit them by hand.

Tips

  • You can start/stop Discovery sessions independently of the packet capture.
  • Loading a PCAP with a Discovery session active will feed those packets into discovery too.
  • The EQ addon requires seeing OP_SessionResponse to learn CRC/encryption params — make sure your capture or PCAP includes connection setup.
  • If using addon mode without a profile selected, Discovery will run the addon's parser internally to build session state.
